一、系统核心业务目标

企税康协同工作平台面向财税/企服机构（销售、顾问、政务经办、财务、人事、管理层），目标是把获客 → 签约 → 履约 → 收款 → 运营管控串成一条协同链路：

目标维度具体诉求
销售增长线索/商机跟进、目标与业绩考核、合同签署与订单收款
客户全生命周期客户主数据、档案维护、物品移交、多业务线服务状态可视
履约交付政务服务工单、代账客户服务、注册地址/房间资源管理
经营与内控合同订单、开票与收款、报销与人事审批、薪酬发放、经营报表
组织与系统员工/部门、角色权限、字典与菜单配置、知识库赋能一线
管理驾驶舱工作台聚合 KPI、待办、排行与业务速览，支撑日常决策

本质定位：B2B 财税服务企业的一站式内部协同 OS，以「客户」为中心，销售与交付双线并行。

二、系统功能模块

按侧栏信息架构（assets/menu-data.js）划分为 7 大模块 + 登录入口：

  1. 入口 — login.html
  2. 工作台 — index.html
  3. 销售管理 — 销售目标、线索、商机、销售业绩（4 页 + 2 详情）
  4. 客户中心 — 客户、政务、代账、地址、客户物品（5 列表 + 5 详情/编辑）
  5. 知识库 — knowledge-base.html
  6. 合同订单 — 合同管理、订单管理
  7. 运营管理 — 财务报销、经营报表、人事管理、薪资管理、员工管理
  8. 系统管理 — 角色权限、字典管理、菜单管理

4.3 销售漏斗链路

工作台录线索 → 线索管理 → [转商机] → 商机管理 → 商机详情 ↓ 创建合同 → 合同管理 → 订单管理 销售目标、销售业绩 ← 考核与排行 ← 工作台工作简报/月度排行
  • 线索 → 商机：lead-management「转商机」
  • 商机/客户 → 合同：共用 create-contract-modal.js
  • 合同 → 订单：order-management 可回链 contract-management#合同号

4.4 客户中心服务链

客户管理（主数据） ├─ 客户详情 / 编辑信息 ├─ 客户物品 ├─ 政务服务 → 工单详情 ├─ 代账服务 → 代账详情 └─ 地址管理 → 地址详情（房间）

4.6 典型业务闭环

成单闭环：线索/商机 → 创建合同 → 订单 → 收款/开票 → 代账/政务履约 → 工作台 KPI/排行

续费闭环：客户详情 ↔ 代账/政务 ↔ 工作台待办（代账到期、续费跟进）
