## 一、系统核心业务目标

**企税康协同工作平台** 面向财税/企服机构（销售、顾问、政务经办、财务、人事、管理层），目标是把 **获客 → 签约 → 履约 → 收款 → 运营管控** 串成一条协同链路：

| 目标维度 | 具体诉求 |
|---------|---------|
| **销售增长** | 线索/商机跟进、目标与业绩考核、合同签署与订单收款 |
| **客户全生命周期** | 客户主数据、档案维护、物品移交、多业务线服务状态可视 |
| **履约交付** | 政务服务工单、代账客户服务、注册地址/房间资源管理 |
| **经营与内控** | 合同订单、开票与收款、报销与人事审批、薪酬发放、经营报表 |
| **组织与系统** | 员工/部门、角色权限、字典与菜单配置、知识库赋能一线 |
| **管理驾驶舱** | 工作台聚合 KPI、待办、排行与业务速览，支撑日常决策 |

**本质定位：** B2B 财税服务企业的一站式内部协同 OS，以「客户」为中心，销售与交付双线并行。

## 二、系统功能模块

按侧栏信息架构（`assets/menu-data.js`）划分为 **7 大模块 + 登录入口**：

1. **入口** — login.html  
2. **工作台** — index.html  
3. **销售管理** — 销售目标、线索、商机、销售业绩（4 页 + 2 详情）  
4. **客户中心** — 客户、政务、代账、地址、客户物品（5 列表 + 5 详情/编辑）  
5. **知识库** — knowledge-base.html  
6. **合同订单** — 合同管理、订单管理  
7. **运营管理** — 财务报销、经营报表、人事管理、薪资管理、员工管理（5 页 + 2 员工详情）  
8. **系统管理** — 角色权限、字典管理、菜单管理

## 四、页面关联关系

### 4.1 主干导航关系

几乎所有业务页共享同一套侧栏信息架构，模块间为平级跳转。详情页通过 `assets/app-shell.js` 中 `DETAIL_PARENT` 在侧栏高亮对应列表页：

| 详情页 | 侧栏高亮父页 |
|--------|-------------|
| customer-detail.html / customer-edit.html | customer-management.html |
| lead-follow-detail.html | lead-management.html |
| opportunity-detail.html | opportunity-management.html |
| government-affairs-detail.html | government-affairs.html |
| accounting-customer-detail.html | accounting-customer-management.html |
| address-room-detail.html | address-management.html |
| employee-profile.html / employee-edit.html | department-employees.html |

### 4.2 列表 → 详情（主链路）

| 列表页 | 详情/编辑页 | URL 参数 |
|--------|-------------|----------|
| customer-management | customer-detail、customer-edit | ?id=cust-n |
| lead-management | lead-follow-detail | ?id=lead-n |
| opportunity-management | opportunity-detail | ?id=opp-n |
| government-affairs | government-affairs-detail | ?id=gov-n |
| accounting-customer-management | accounting-customer-detail | ?id=acc-n |
| address-management | address-room-detail | 无 query |
| department-employees | employee-profile、employee-edit | ?name= 或静态链 |

### 4.3 销售漏斗链路

```
工作台录线索 → 线索管理 → [转商机] → 商机管理 → 商机详情
                                    ↓
                              创建合同 → 合同管理 → 订单管理
销售目标、销售业绩 ← 考核与排行反馈 ← 工作台工作简报/月度排行
```

- 线索 → 商机：lead-management 内「转商机」
- 商机/客户 → 合同：opportunity-management、customer-management、工作台 → 共用 create-contract-modal.js
- 合同 → 订单：order-management 可经 contract-management.html#合同号 回链

### 4.4 客户中心服务链

```
客户管理（主数据）
  ├─ 客户详情 / 编辑信息
  ├─ 客户物品
  ├─ 政务服务 → 工单详情
  ├─ 代账服务 → 代账详情
  └─ 地址管理 → 地址详情（房间）
```

客户是主实体；政务、代账、地址、物品是不同服务线/资源线。工作台「业务速览」汇总政务/代账等待办类指标。

### 4.5 工作台枢纽

| 工作台能力 | 关联页面 |
|-----------|----------|
| 快捷录线索/跟进/合同/维护 | 弹窗或跳转销售/客户模块 |
| 工作简报指标 | 线索、商机、合同、订单、政务、代账、地址等业务域 |
| 待办事项 | 合同、审批、回访、资料、续费等 |
| 月度业绩排行 → 更多 | sales-performance.html |
| 知识库摘要 | knowledge-base.html |

### 4.6 运营与组织

```
员工管理 → 员工档案 → 员工信息编辑
    ↓
人事管理、薪资管理
角色权限 → 控制全站各模块操作
字典管理、菜单管理 → 全站枚举与导航配置
```

### 4.7 典型业务闭环

**成单闭环：**  
线索/商机 → 创建合同 → 订单 → 收款/开票（订单内） → 代账/政务派单履约 → 工作台 KPI/业绩排行反馈

**续费/服务闭环：**  
客户详情服务详情 ↔ 代账/政务 ↔ 工作台「代账到期」「续费跟进」待办
